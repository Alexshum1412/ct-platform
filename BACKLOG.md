# CT-Platform — бэклог будущих дел

Обновлено: 12 июня 2026. Приоритеты: 🔴 критично для продакшена · 🟡 важно · 🟢 желательно.

## Инфраструктура и продакшен

- 🔴 **SMTP для писем.** Подтверждение email и восстановление пароля сейчас работают через dev-код в ответе API (фолбэк при пустом `SMTP_USER`). Настроить реальный SMTP (см. `EMAIL_VERIFICATION_SETUP.md`).
- 🔴 **Платёжный провайдер.** `PaymentPage` — визуальный стаб; Premium выдаётся только при `ALLOW_DEMO_PREMIUM=true`. Интегрировать реальные платежи (bePaid/WebPay/ЕРИП) с server-side валидацией до активации подписки.
- 🟡 **Серверный контроль длительности экзамена.** Сейчас таймер только на клиенте; сабмит после истечения времени сервер не отклоняет. Хранить `startedAt` + `durationMinutes` и проверять на `/api/exam/submit`.
- 🟡 **`correctAnswer` в выдаче практики.** Осознанный трейдофф ради мгновенной проверки, но ответ виден в network-вкладке. Рассмотреть переход на серверную проверку ответа практики.
- 🟡 **Миграции Prisma.** Схема развивается через `db push`; `migrations/0_init` отстал. Сгенерировать свежий baseline, чтобы новые окружения поднимались `migrate deploy` без сюрпризов.
- 🟢 **Redis для rate-limit** при масштабировании на несколько инстансов (сейчас in-memory фолбэк).
- 🟢 **Мониторинг:** Sentry (фронт+бэк), алёрты на 5xx Render, аптайм-чекер.
- 🟢 **Audit log retention:** таблица `audit_logs` будет расти — добавить автоочистку (например, >12 мес) или архивирование.

## Контент

- 🔴 **Реальные уровни сложности вопросов.** Все 1984+ заданий сейчас `difficulty=1` (после массового фикса) — это ломает стратификацию вариантов ЦТ ({1:5,2:10,3:9,4:4,5:2}). Переразметить сложность, затем перегенерировать варианты `seed-ct-math.js --variants=5`.
- 🟡 **Банк вопросов гуманитарных предметов.** Русский 32, история/обществоведение/языки по 7–10 вопросов — мало для практики. Нужен ручной контент или новые генераторы.
- 🟡 **Олимпиады, волна 3.** После волны 2 у 7 «малых» предметов по 2 задачи REGION/REPUBLIC — догнать до 4–5; добавить теорию PRO (сейчас 11 статей на 11 предметов).
- 🟡 **Авторские материалы теории.** JSON-поля `formulas`/`examples` пустые; добавить разобранные примеры и exam-traps по ключевым темам.
- 🟢 **Реальный архив олимпиад** прошлых лет с указанием источников (сейчас тренировочные задачи «в формате олимпиады»).
- 🟢 **562 теории / 401 подтема без вопросов** — контентный долг по покрытию практикой.

## Функциональность

- 🟡 **Уведомления.** Модель `Notification` есть, но нет ни UI-центра уведомлений, ни доставки (email/push). Реализовать или убрать модель.
- 🟡 **Bulk-операции для теории и экзаменов** в админке (сейчас только задания и олимпиадные задачи).
- 🟡 **Роль MODERATOR** — сейчас только ADMIN/USER; модератору можно отдать «На проверке», «Жалобы», «Сообщения» без полного доступа.
- 🟢 **Мягкое удаление (undo)** для контента в админке — сейчас удаление необратимо (журнал хранит снимок, но восстановления нет).
- 🟢 **Олимпиадный «тур на время»** — полный комплект задач этапа с таймером и оформлением, как на реальном туре.
- 🟢 **Повторение ошибок (spaced repetition)** — авто-подборка из ранее неверно решённых заданий по кривой забывания.
- 🟢 **Глобальный поиск** по заданиям/теории/олимпиадам из хедера.
- 🟢 **Группы/классы для учителей** (b2b): учитель видит прогресс учеников, назначает домашние подборки.
- 🟢 **Экспорт прогресса** в PDF (отчёт для родителей/учителя).
- 🟢 **Comments API** — спящий код без UI: включить (комментарии к заданиям) или удалить.

## UI/UX и качество

- 🟡 **Перфоманс: KaTeX-чанк 265 KB** (gzip 77 KB) — самый тяжёлый; грузить лениво только на страницах с формулами.
- 🟡 **a11y-аудит:** фокус-треп в кастомных оверлеях (delete-confirm в профиле), aria-атрибуты у самодельных селектов, контраст muted-текста в тёмной теме.
- 🟢 **e2e-тесты (Playwright)** поверх API-smoke: регистрация→верификация→практика→экзамен→профиль; админ-CRUD; мобильный вьюпорт.
- 🟢 **Тёмная тема:** точечный аудит новых блоков (audit log, bulk-панели, hero-карточки) на контраст.
- 🟢 **Скелетоны** для AuditLogViewer-фильтров и олимпиадного хаба при медленной сети (частично есть).
- 🟢 **PWA/offline** для раздела теории.
