## Вход в админ-панель

### Требования

- Аккаунт с ролью `ADMIN` в базе данных
- Доступ к панели по адресу: `https://your-domain.com/admin`

### Процесс входа

1. Авторизуйтесь на платформе с админским аккаунтом
2. Перейдите по ссылке `/admin` или выберите "Админ-панель" в меню пользователя
3. Система автоматически проверит вашу роль и предоставит доступ

### Назначение администратора

```sql
-- В базе данных PostgreSQL
UPDATE "User" SET role = 'ADMIN' WHERE email = 'admin@example.com';
```

## Обзор интерфейса

Админ-панель состоит из 4 основных разделов:

### 1. Дашборд (Статистика)

Отображает ключевые метрики платформы:
- **Пользователей** — общее количество зарегистрированных пользователей
- **Заданий** — количество вопросов в базе
- **Решений** — общее количество ответов пользователей
- **На проверке** — задания, ожидающие модерации
- **Новых сегодня** — пользователи, зарегистрировавшиеся сегодня
- **Активных** — пользователи онлайн

### 2. Вкладки навигации

- **Задания** — управление вопросами (CRUD)
- **На проверке** — модерация пользовательского контента
- **Пользователи** — управление пользователями
- **Статистика** — детальная аналитика

## Управление заданиями

### Просмотр списка заданий

1. Перейдите на вкладку "Задания"
2. Используйте **поиск** для фильтрации по содержимому
3. Используйте **выпадающий список** для фильтрации по предмету

### Добавление нового задания

1. Нажмите кнопку **"Добавить задание"**
2. Заполните форму:
   - **Предмет** — выберите из списка
   - **Тема** — выберите или создайте новую
   - **Тип задания** — SINGLE_CHOICE, MULTIPLE_CHOICE, TEXT_INPUT
   - **Сложность** — от 1 (лёгкий) до 5 (эксперт)
   - **Содержание** — текст вопроса с поддержкой LaTeX (`$...$`)
   - **Варианты ответов** — для выборочных заданий
   - **Правильный ответ** — ID правильного варианта или текст
   - **Объяснение** — разбор ответа с поддержкой LaTeX
   - **Подсказки** — три уровня подсказок
3. Нажмите **"Сохранить"**

### Редактирование задания

1. Найдите задание в списке
2. Нажмите иконку **карандаша** (редактировать)
3. Внесите изменения
4. Нажмите **"Сохранить"**

### Удаление задания

1. Найдите задание в списке
2. Нажмите иконку **корзины** (удалить)
3. Подтвердите удаление в диалоговом окне

**Внимание:** Удаление невозможно отменить!

-- Блочная формула

## Модерация контента

### Просмотр заданий на проверке

1. Перейдите на вкладку "На проверке"
2. В списке отображаются задания, предложенные пользователями

### Одобрение задания

1. Просмотрите содержимое задания
2. Нажмите **"Одобрить"**
3. Задание появится в общей базе

### Отклонение задания

1. Просмотрите содержимое задания
2. Нажмите **"Отклонить"**
3. Задание будет удалено из очереди

## Управление пользователями

### Просмотр списка пользователей

1. Перейдите на вкладку "Пользователи"
2. Используйте поиск по имени или email

- **Просмотр профиля** — детальная информация

## API Endpoints

### Аутентификация

Все запросы к админ API требуют JWT токен в заголовке:

```http
Authorization: Bearer <token>
```

### Endpoints

#### Статистика

```http
GET /api/admin/stats
```

Ответ:
```json
{
  "totalUsers": 12450,
  "totalQuestions": 5234,
  "totalSolved": 89234,
  "pendingReview": 12,
  "newUsersToday": 45,
  "activeUsers": 1234
}
```

#### Список вопросов

```http
GET /api/admin/questions?subjectId=math&search=дискриминант&limit=20&offset=0
```

#### Создание вопроса

```http
POST /api/admin/questions
Content-Type: application/json

{
  "subjectId": "math",
  "topicId": "algebra",
  "type": "SINGLE_CHOICE",
  "difficulty": 2,
  "content": "Решите $x^2 - 4 = 0$",
  "options": [
    { "id": "a", "text": "$x = 2$" },
    { "id": "b", "text": "$x = -2$" },
    { "id": "c", "text": "$x = \pm 2$" }
  ],
  "correctAnswer": "c",
  "explanation": "$x^2 = 4$, значит $x = \pm 2$",
  "tags": ["квадратное уравнение"]
}
```

#### Обновление вопроса

```http
PUT /api/admin/questions/:id
Content-Type: application/json

{
  "difficulty": 3,
  "content": "Обновлённый текст"
}
```

#### Удаление вопроса

```http
DELETE /api/admin/questions/:id
```

#### Вопросы на модерации

```http
GET /api/admin/pending
```

#### Одобрить вопрос

```http
POST /api/admin/pending/:id/approve
```

#### Отклонить вопрос

```http
POST /api/admin/pending/:id/reject
```

## Решение проблем

### Нет доступа к админ-панели

**Проблема:** При переходе на `/admin` показывается "Доступ запрещён"

**Решение:**
1. Проверьте, что вы авторизованы
2. Проверьте роль пользователя в БД:
   ```sql
   SELECT email, role FROM "User" WHERE email = 'your@email.com';
   ```
3. Убедитесь, что `role = 'ADMIN'`

### Данные не загружаются

**Проблема:** Статистика или список заданий пустые

**Решение:**
1. Проверьте подключение к backend API
2. Проверьте JWT токен в localStorage
3. Посмотрите ошибки в консоли браузера (F12)
4. Проверьте логи сервера

### Ошибка при сохранении задания

**Проблема:** Форма не сохраняется

**Решение:**
1. Проверьте обязательные поля (предмет, содержание, ответ)
2. Убедитесь, что JSON валиден
3. Проверьте консоль на ошибки валидации

## Безопасность

### Рекомендации

1. **Используйте сильные пароли** для админских аккаунтов
2. **Включите 2FA** если доступно
3. **Не передавайте доступ** третьим лицам
4. **Регулярно проверяйте логи** на подозрительную активность
5. **Выходите из системы** после завершения работы

### Мониторинг

Следите за:
- Необычными пиками активности
- Множественными неудачными попытками входа
- Подозрительными действиями пользователей
